Bosco Dialer

Call queue + Aircall auto-dialer

Open control panel

OPEN CONTROL PANEL →

Tailscale IP (100.x.x.x) works from anywhere, including cell data — just have Tailscale on. A LAN IP (192.168.x.x) only works on the same Wi-Fi. Saved on this device, so next time just tap the button.

The control panel runs on your PC, not here — this page just launches it. The bridge must be running, and your phone needs a route to the PC (Tailscale on, or same Wi-Fi).

Install / update the userscript

INSTALL USERSCRIPT

Needs Tampermonkey. After installing once, it auto-updates from this repo — no more copy-pasting.

Sod texter (A/B campaign)

INSTALL SOD TEXTER

Separate script. Reuses the dialer's scan to grab Tech Notes flagged sod webworm (skips anyone already on surface insecticide), splits 50/50 price vs no-price, previews, then texts through the bridge. Keeps a permanent list so nobody is texted twice. Panel appears bottom-left on the call log. Run the dialer scan first so its results are ready.

Keys

↑Answered — then pick GO NEXT or RESOLVE
↓No answer — log note, reschedule (or resolve on the 2nd)
rResolve — "not interested in <treatment>" note, then close
hHold — pause without hanging up the call
EscPause (hangs up) / resume (redials same lead)
EnterStart calling the queue
fRescan the call log from scratch
sCopy the "not calling" log
cClear everything (wipes the saved queue)

Arrows work anywhere on screen while the bridge is running.

Daily start-up

  1. Open the Bosco call-log tab.
  2. Double-click start-dialer.bat — leave the black window open.
  3. Panel header in Bosco turns blue = bridge connected.
  4. It loads the saved queue and starts calling (f to rescan).

Phone control needs setup-phone.bat run as administrator once.
